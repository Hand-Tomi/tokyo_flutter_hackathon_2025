# :memo: 제품 요구사항 정의서 (해커톤 MVP)

---

## 1. 목표 및 비전

* **핵심 컨셉:** 부모의 음성과 아이의 Air Scribble 제스처로 장면(Scene)을 생성하고, 이를 합쳐 **AI 동화 영상(MP4)**을 자동 제작한다.
* **핵심 가치:** 부모의 목소리 + 아이의 움직임이 담긴 세상에 하나뿐인 창작물
* **해커톤 목표:** 최소 기능 기반 빠른 구현, 자연스러운 데모 중심

---

## 2. 사용자 경험 시나리오

1. 부모가 한 장면의 내용을 음성으로 말한다.
2. 시스템이 자동으로 STT 텍스트를 생성한다. (수정 없음)
3. 아이가 허공에서 손 제스처(Air Scribble)를 그린다.
4. 텍스트 + Scribble로 장면(Scene) 1개 생성 → 백그라운드에서 이미지 자동 생성
5. “다음 장면 만들기”를 선택하여 반복
6. “완료” 시, 생성된 장면들을 합쳐 최종 동화 영상을 생성한다.

---

## 3. 핵심 기능 요구사항 (MVP 중심)

### 3.1 음성 → STT

* 부모 음성 녹음 기능
* STT 자동 변환
* 텍스트 수정 불가(그대로 사용)
* STT 결과는 장면 설명(prompt)으로 사용

### 3.2 Air Scribble 제스처 입력

* 카메라 기반 제스처 추적 → 단순 2D 라인 생성
* 실시간 프리뷰 제공
* **Clear 버튼으로 전체 삭제 후 다시 그리기 가능**
* Undo는 선택 사항 (없어도 MVP OK)

### 3.3 AI 장면 이미지 생성

* ControlNet Scribble 기반 단일 이미지 생성
* 스타일 1종 고정
* 입력: STT 텍스트 + Scribble
* 백그라운드에서 생성하여 사용자 흐름 방해 X

### 3.4 영상 생성 (슬라이드쇼 기반)

* 장면 이미지들을 순서대로 연결
* 부모 음성(장면별)을 하나로 이어붙여 타임라인 구성
* 간단한 페이드 전환
* 최종 MP4 출력

---

## 4. 화면 구성 (Screens)

1. **홈 화면** – 시작하기 / 기존 작품 보기
2. **장면 생성 통합 화면** *(음성 + STT + Air Scribble)*

   * 음성 녹음 → STT 자동 생성 → Air Scribble → 장면 확정
   * 장면 확정 후 바로 다음 장면 입력 가능 (이미지는 백그라운드 생성)
   * Clear 버튼 제공
   * "장면 생성 종료" 버튼으로 리스트로 이동
3. **장면 리스트 화면**

   * 생성된 장면 썸네일 (생성 중인 경우 로딩 표시)
   * 삭제 기능 (선택)
   * "영상 만들기" 버튼
4. **영상 생성 화면**

   * 장면 합성 중 로딩 표시
5. **완성된 영상 재생 화면**

   * 최종 MP4 재생
6. **저장·공유 화면**

   * 다운로드 / 공유 기능

---

## 5. 화면 흐름도

```
[홈]
   ↓
[장면 생성 통합 화면]
   - 음성 녹음 → STT → Air Scribble → 장면 확정(즉시 다음 장면)
   - 장면 이미지는 백그라운드 생성
   ↓
[장면 생성 종료]
   ↓
[장면 리스트]
   ↓
[영상 생성]
   ↓
[완성된 영상 재생]
   ↓
[저장·공유]
```

---

## 6. 비기능 요구사항

* **성능:** 장면 이미지 생성 ≤ 30초 (백그라운드 처리)
* **사용성:** 부모·아이 모두 쉽게 사용할 수 있는 단순 UI
* **개발 난이도:** 1~2일 내 구현 가능한 기능만 포함
* **보안:** 음성·제스처 데이터는 안전하게 처리
* **호환성:** 웹 기반 또는 단일 플랫폼 기준 구현

---

## 7. 해커톤 범위 요약

### 필수 (Must)

* 음성 녹음 + STT 자동 생성
* Air Scribble → 2D 라인
* Clear 기능
* 장면 반복 생성 루프
* 백그라운드 AI 이미지 생성
* 이미지 + 음성 기반 영상(MP4) 자동 생성
* 영상 재생 + 다운로드

### 선택 (Nice)

* 장면 재정렬
* 간단한 전환효과 강화
* 생성 중 썸네일 프리뷰
